Strategic RBAC — User Stories
BT Nexus — Strategic RBAC
User Stories
Platform: Salesforce RBAC (confirmed)
Model: Permission Sets → Roles → Users
Status: Draft for BA review · July 2026
Priority
Persona
42 of 42 stories
Priority (MoSCoW)
Must — core RBAC functionality, required for launch
Should — important but not launch-blocking
Could — valuable if capacity allows
Context
Identity platform: Salesforce RBAC. Org hierarchy: BT → Reseller → Sub-reseller → Customer. Permission sets form roles; roles are assigned to users.
Organisation Hierarchy & Access Scoping
RBAC-01 Must
As a BT Wholesale admin, I want to define the organisation hierarchy (Reseller → Sub-reseller → Customer) so that access and data are scoped to the correct tier automatically.
BT Admin
RBAC-02 Must
As a reseller admin, I want to see only the organisations and users within my own hierarchy so that I cannot view or affect data belonging to other resellers.
Org Admin
RBAC-03 Must
As a BT Wholesale admin, I want to view and navigate the full organisation tree so that I can provide support to any organisation in the network without logging into each one separately.
BT Admin
RBAC-04 Should
As a sub-reseller admin, I want to manage only my own downstream customers so that my administrative scope is contained to my part of the network.
Org Admin
RBAC-05 Should
As a BT Wholesale admin, I want to assign an organisation type at provisioning time so that the correct permission boundaries are applied from day one, without manual configuration after the fact.
BT Admin
RBAC-06 Could
As a reseller admin, I want a summary view of role coverage and access gaps across my downstream orgs so that I can proactively identify risks without drilling into each organisation individually.
Org Admin
Role Management
RBAC-07 Must
As an org admin, I want to assign an existing role to a user so that they gain the correct access for their responsibilities without me having to configure permissions from scratch.
Org Admin
RBAC-08 Must
As an org admin, I want to view all roles available in my organisation and what each one covers so that I can make informed assignment decisions.
Org Admin
RBAC-09 Should
As an org admin, I want to create a custom role by selecting permission sets from the catalogue so that I can match access to how my team actually works rather than fitting people into standard roles.
Org Admin
Depends on Salesforce RBAC supporting custom permission set composition. Confirm with Vanmathi.
RBAC-10 Should
As an org admin, I want to edit the permission sets included in an existing role so that I can adjust access without rebuilding roles from scratch when responsibilities shift.
Org Admin
RBAC-11 Should
As a BT Wholesale admin, I want to define a library of standard roles so that resellers have sensible, named defaults to start from rather than composing access from scratch.
BT Admin
RBAC-12 Could
As an org admin, I want to clone an existing role as the starting point for a new one so that I don't have to select permission sets from scratch when creating a role that's a variation of an existing one.
Org Admin
RBAC-13 Could
As an org admin, I want to deactivate a role so that it can no longer be assigned, without losing historical records of who held it and when.
Org Admin
Permission Sets
RBAC-14 Must
As a BT Wholesale admin, I want to define and maintain a catalogue of permission sets grouped by capability area (Orders, Faults, Billing, Users & Orgs, Reporting, API, Support) so that roles can be composed granularly from named, documented capabilities.
BT Admin
RBAC-15 Must
As an org admin, I want to view the full permission set catalogue with plain-English descriptions so that I understand exactly what each set grants before adding it to a role.
Org Admin
RBAC-16 Should
As an org admin, I want to add a permission set to a role so that I can extend what that role can do without creating an entirely new role.
Org Admin
RBAC-17 Should
As an org admin, I want to remove a permission set from a role so that I can reduce access when a team member's responsibilities change.
Org Admin
RBAC-18 Should
As a user, I want to see which permission sets my assigned role includes so that I understand what I can and cannot access before attempting to use a feature.
End User
RBAC-19 Could
As a BT Wholesale admin, I want to restrict which permission sets are available per organisation type so that resellers cannot grant capabilities that BT has not enabled for their tier.
BT Admin
User Onboarding & Invitation
RBAC-20 Must
As an org admin, I want to invite a new user by email address so that they can join my organisation and gain platform access without me having to raise a request with BT.
Org Admin
RBAC-21 Must
As an org admin, I want to assign a role at the point of invitation so that the user has the correct access from their very first login, not after a separate step.
Org Admin
RBAC-22 Must
As an invited user, I want to receive a secure, time-limited email invitation so that I can set up my account safely and the link cannot be used by anyone else.
End User
RBAC-23 Must
As a new user, I want to complete identity verification during onboarding via Salesforce Identity so that my account meets BT Wholesale's security and compliance requirements before I gain access.
End User
Identity platform confirmed as Salesforce RBAC (July 2026). Strategic Identity ruled out. Vanmathi to confirm exact verification flow.
RBAC-24 Should
As an org admin, I want to see the status of all outstanding invitations (pending, accepted, expired) so that I can chase or resend without losing track of who still needs to onboard.
Org Admin
RBAC-25 Should
As an org admin, I want to cancel a pending invitation so that I can prevent access immediately if circumstances change before the user has accepted.
Org Admin
User Management
RBAC-26 Must
As an org admin, I want to view all users in my organisation with their current role and status so that I have a clear picture of who has access to what at any point in time.
Org Admin
RBAC-27 Must
As an org admin, I want to change a user's role so that their access accurately reflects their current responsibilities when their job changes.
Org Admin
RBAC-28 Must
As an org admin, I want to deactivate a user so that they immediately and completely lose platform access — for example when someone leaves the organisation.
Org Admin
RBAC-29 Should
As an org admin, I want to reactivate a deactivated user without re-inviting them so that I can restore access efficiently if someone returns or was deactivated in error.
Org Admin
RBAC-30 Should
As an org admin, I want to filter and search users by name, role, or status so that I can find the right person quickly in organisations with large user lists.
Org Admin
RBAC-31 Could
As an org admin, I want to change the role for multiple users at once so that I can handle team restructures without updating each person individually.
Org Admin
RBAC-32 Could
As a user, I want to request a role change through the portal so that my admin can review and approve it without an out-of-band email or phone call.
End User
API Access & Product Entitlements
RBAC-33 Must
As a BT Wholesale admin, I want to define which products and journeys are enabled per organisation so that product entitlements are enforced at the RBAC layer, not just hidden in the UI.
BT Admin
RBAC-34 Must
As an API developer, I want to generate an API key that inherits my role's permission scope so that my integration cannot access data or actions beyond what my role allows.
API Developer
RBAC-35 Should
As an org admin, I want to create, rotate, and revoke API keys on behalf of developers in my organisation so that I maintain oversight of API access and can act quickly if a key is compromised.
Org Admin
RBAC-36 Should
As an org admin, I want to see which products and journeys are enabled for my organisation so that I understand what capabilities my users can access before assigning roles.
Org Admin
RBAC-37 Could
As a BT Wholesale admin, I want RBAC enforcement to apply consistently across the portal UI, API, and backend services so that there is no path — technical or otherwise — to bypass a permission restriction.
BT Admin
Listed as a PRD NFR. Surfaced here because it has significant implementation implications across all layers.
Audit & Compliance
RBAC-38 Must
As a BT Wholesale admin, I want a tamper-evident audit log of all role assignments, changes, and revocations so that I can investigate access incidents and demonstrate compliance.
BT Admin
RBAC-39 Should
As an org admin, I want to view a log of invitation and onboarding activity in my organisation so that I have a record of who joined, when they were invited, and who approved their access.
Org Admin
RBAC-40 Should
As an org admin, I want to see a full role history for any user in my organisation so that I can understand how their access has changed over time during an access review.
Org Admin
RBAC-41 Should
As a user, I want to receive a notification when my role is changed so that I'm always aware of changes to what I can access, and can flag anything that seems wrong.
End User
RBAC-42 Could
As a BT Wholesale admin, I want to export audit logs in a structured format so that they can be ingested by external compliance or SIEM tooling without manual processing.
BT Admin
