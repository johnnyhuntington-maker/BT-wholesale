Persona Cards
8 personas across 4 org contexts · Source: Jason Grant briefing (14 Jul 2026) + PRD
BT Wholesale — internal
AM
BT Internal Admin
Platform Administrator · BT Wholesale
Governs the entire reseller network. Sets up reseller orgs, assigns entitlements, and defines platform-wide access rules.
Can
  • Create and manage reseller organisations
  • Assign products and entitlements to resellers
  • Set up reseller administrators
  • Define platform-wide access rules
Cannot
  • Carry out a reseller's day-to-day order management
  • Register or onboard a new organisation directly
Prototype: Alex Morgan
CA
BT Account Manager / Support
Account Manager · BT Wholesale
Manages BT Wholesale's relationships with reseller partners. Views entitlements and assists onboarding — but platform settings are for the Platform Admin.
Can
  • View and manage reseller organisation profiles
  • Support reseller onboarding and account setup
  • Review entitlements across the reseller network
  • Assist resellers with user and role queries
Cannot
  • Create new reseller organisations
  • Change platform-wide access rules
Prototype: Claire Ashton
Reseller — direct BT contract
SW
Reseller Admin
Administrator · Northgate Telecom
Runs the reseller account. Manages users, builds the downstream network, and controls what sub-orgs can access.
Can
  • Create sub-reseller, child reseller & dealer orgs
  • Invite team members and assign roles
  • Control what downstream orgs can access
  • Full billing, ordering, support and API access
Cannot
  • Access products not granted by BT Wholesale
  • Change BT platform-level settings
Prototype: Sarah Whitfield
JO
Reseller User
Scoped to assigned role · Northgate Telecom
Staff member at a reseller. Accesses only what their assigned role permits — no user management or org-level control.
Can
  • Access portal tools scoped to their role
  • Place orders / raise faults / view reports (role-dependent)
  • View own permissions in their profile
Cannot
  • Create or manage organisations
  • Invite or manage other users
  • Change their own role or permissions
Prototype: James Okafor (Order Manager)
Sub-reseller — no direct BT contract
MW
Sub-Reseller Admin
Administrator · Metro Connect
Runs the sub-reseller account under Northgate Telecom. Manages team and places orders, but restricted from billing, support and hardware.
Can
  • Invite team members and assign roles
  • Place and manage orders
  • Choose which services to expose to users
  • Receive customer KCIs
Cannot
  • Create child organisations
  • Access billing or commercial data (PRD: RBAC-enforced)
  • Raise support tickets with BT
  • Hardware ordering, Business Zone, FMS/Empirix
Prototype: Marcus Webb
SR
Sub-Reseller User
Scoped to assigned role · Metro Connect
Staff at a sub-reseller. Same pattern as Reseller User but within the reduced capability envelope of the sub-reseller org.
Can
  • Access tools scoped to their role (if enabled by sub-reseller admin)
  • Place orders where permitted
  • Track faults and appointments
Cannot
  • Access anything the sub-reseller org doesn't have
  • Manage users, orgs or billing
Not yet in prototype
API portal — developer track
AS
API Supplier
Supplies capability via API · External
Provides services or capability that BT Wholesale exposes through the API platform. A producer, not a consumer of the portal.
Can
  • Publish API capabilities to the BT Wholesale platform
  • Manage their own product API specs
Not yet in prototype · Matthew Lindley owns
PD
API Consumer / Partner Developer
Builds on BT APIs · Wholesale Partner
A developer at a wholesale partner who integrates BT Wholesale services directly into their own systems via API. Needs sandbox and production API access.
Can
  • Authenticate and call Nexus APIs
  • Access sandbox environment for testing
  • Create and manage API keys for their org
  • View product and order data via API
Cannot
  • Manage users or org settings
  • Access billing or support journeys (unless role permits)
Prototype role: API Developer
